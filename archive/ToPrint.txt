Part 4: The "Context-Slice-Test-Ship" Mantra
(Print this out or keep it on a sticky note next to your monitor)

When you open Claude Code, you are the Manager. Claude is the Junior Developer.

Step 1: CONTEXT (Anchor the AI)

Prompt: "Read AGENTS.md. Read .agent/skills/[relevant-skill].md. Read docs/specs/phase-X/[spec-name].md."

Why: If you don't do this, it guesses.

Step 2: SLICE (Scope the task)

Prompt: "We are executing AC-1 (Acceptance Criteria 1) from the spec. Write the SQLAlchemy model and the Pytest test for it. Do not write the API route yet."

Why: Constrains the LLM so it doesn't break 5 files at once.

Step 3: TEST (Prove it works)

Prompt: "Run python -m pytest tests/test_[name].py -v. Fix any errors."

The Loop Breaker Rule: If Claude fails the test 3 times in a row, type "STOP." Read the error yourself. The AI is stuck in a hallucination loop. Point it in the right direction.

Step 4: SHIP (Lock it in)

Prompt: "The tests passed. Run git add -A and git commit -m 'feat: [description]'."

Why: Save your progress. If the next slice breaks everything, you just git reset --hard and you only lose 20 minutes of work, not 4 hours.
