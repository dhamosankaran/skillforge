# SkillForge Backlog

> Single source of truth for all work items: bugs, enhancements, and tech debt.
> Spec files in `docs/specs/` describe the *what* and *how*. This file tracks the *what's-next* and *what's-done*.

## Rules
- **IDs are immutable.** Once assigned, a B-### or E-### never changes, never gets reused.
- **One row per item.** Don't split, don't merge. Spawn a new ID if scope splinters.
- **Status is the only field Claude Code may flip autonomously** (ЁЯФ┤ тЖТ ЁЯЯб тЖТ тЬЕ). All other fields require human edit.
- **Closing an item:** status тЖТ тЬЕ + append a one-liner to Notes: `closed by <commit-sha> on <YYYY-MM-DD>`.
- **Priority is a product decision.** Set by Dhamo only.
- **Spec column is the on-disk path** (`docs/specs/phase-N/NN-name.md`). Empty if no spec exists yet тАФ create one before implementation.

## Status legend
- ЁЯФ┤ Pending (not started)
- ЁЯЯб In progress / partial / blocked
- тЬЕ Done
- тЪк Won't do / deferred indefinitely
- ЁЯЯж Back-burner тАФ scoped out, awaiting activation gate (e.g., telemetry threshold). Not worked on until the gate trips.

- **P0** тАФ broken in production, blocking users, blocking next milestone
- **P1** тАФ committed for this phase
- **P2** тАФ wanted, not committed
